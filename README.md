# 契約管理システム (Contract Management System)

次回参画案件の事前学習および実務シミュレーションを目的として開発した、Java（Struts1）とOracle DBを連携させたWebアプリケーションです。

---

## 📄 開発背景 / プロジェクトの目的

次回参画予定の案件において、**Java（Struts1）および Oracle DB** を使用した既存の基幹システムの保守・機能追加を担当することが決定したため、その**先回りした技術キャッチアップ（事前学習）**を目的として本システムを完全自作しました。

単に「動くコードを書く」だけでなく、実際の開発現場のチーム開発に即座に馴染めるよう、自分自身に**「実務水準の制約」**を課し、セキュリティ対策やGit運用まで徹底してシミュレーションを行い開発を完了させました。

## 💡 本プロジェクトにおける「実務水準」のこだわり

### 1. 現場のルールに即した Git / GitHub ブランチ運用
1人での個人開発であっても、実務のチーム開発のワークフローを100%シミュレーションしました。
* **Git Flowベースの運用:** 全ての開発を `main` ブランチで行うのではなく、機能ごとに `feat/input-page` や `feat/result-page` といった作業ブランチを必ず作成。
* **プルリクエスト（PR）の活用:** コード完了後はGitHub上でプルリクエストを作成し、差分チェック（セルフレビュー）を実施。説明文には「実施事項」「動作確認結果」を実務レベルのフォーマットで明記。
* **リモート・ローカルのクリーンアップ:** マージ完了後は、役目を終えた作業ブランチを即座に削除し、リポジトリを常に本番同様のクリーンな状態に維持。

### 2. 設定ファイルの分離とセキュリティ対策（機密情報の隠蔽）
実務において最も重大なインシデントの一つである「機密情報の漏洩」を確実に防ぐ設計を行いました。
* **設定ファイル（`.properties`）の外出し:** データベースの接続ユーザー名やパスワードといった重要な環境依存情報をJavaのソースコードから完全に隔離し、`src/main/resources/db.properties` に外出ししました。
* **`.gitignore` によるGit管理除外:** 上記の `db.properties` を `.gitignore` に登録することで、GitHubへのパスワード流出リスクを根本から遮断しました。
* **ソースフォルダの適切なビルドパス設定:** EclipseのJavaビルド・パスを変更し、`resources` を通常のフォルダから「ソースフォルダ」へと昇格。プログラム側（`ResourceBundle`）からは安全かつクリーンに設定ファイルを動的に読み込める実務共通の構成を採用しました。

## 🔍 機能概要

### 1. 契約者情報入力機能
* **Frontend:** Struts1のタグライブラリ（`<html:form>`、`<html:text>` 等）を用いて、業務システムとして標準的な入力フォーム（JSP）を構築。
* **Validation / Data Binding:** 画面からの入力値を保持する `ActionForm` を実装し、リクエストデータをJavaオブジェクトへ安全にバインド。

### 2. データベース登録処理（Backend）
* **Controller (Action):** `Action` クラスがビジネスロジックを制御し、画面からのデータをDAOへ引き渡し、処理結果に応じて最適な遷移先をハンドリング。
* **DAO / SQL 実行:** `PreparedStatement` を使用した強固なDAOクラスを実装。プレースホルダー（`?`）を活用することで、検索・登録時における **SQLインジェクション脆弱性を徹底的に排除**。Oracle DBへの適切なインサートとコミット処理を実現。

### 3. 登録結果（完了）画面への遷移
* **ルーティング制御:** `struts-config.xml` において、処理成功時の転送先（`<forward name="success" path="/success.jsp" />`）を定義。
* アクション処理が正常終了した後に自動で完了画面へ安全にユーザーを誘導し、重複登録（リロードによる二重送信）を防ぐ導線を設計。
